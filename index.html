Git and GitHub

For Beginners

The Problem

  • Lost code changes
  • Overwritten work
  • Who changed what and when?
  • Multiple file versions:
    project_final.zip
    project_final_v2.zip
  • Difficult team collaboration

Version Control System

A system that tracks changes to files

  • See file history
  • Restore old versions
  • Track who made changes
  • Work together without conflicts

Git vs GitHub

Git

  • Software on your computer
  • Tracks file changes
  • Works offline

GitHub

  • Website
  • Stores Git repositories
  • Collaboration platform

Git = the tool, GitHub = the platform

What is Git?

  • Tracks code changes
  • Allows multiple people to work together
  • Records history of your project
  • Works without internet

Repositories

A repository (or "repo") is your project folder

Local Repository

  • On your computer
  • Where you work

Remote Repository

  • On GitHub
  • Shared with others

Branch

A separate version of your project

  • Main branch = official version
  • New branches = working on features
  • Experiment without breaking things

$ git branch feature-login

$ git checkout feature-login

Commit

A snapshot of your changes

  • Saves your work
  • Has a message describing changes
  • Creates project history

$ git add index.html

$ git commit -m "Add login form"

Push

Uploading your commits to GitHub

  • Shares your changes with others
  • Backs up your work online

$ git push origin feature-login

Pull

Downloading changes from GitHub

  • Gets updates from teammates
  • Keeps your local copy up-to-date

$ git pull origin main

Merge

Combining changes from different branches

  • Brings features into the main branch
  • Combines different work together

$ git checkout main

$ git merge feature-login

GitHub Workflow

  1. Create/clone a repository
  2. Create a branch
  3. Make changes
  4. Push your branch
  5. Create a Pull Request
  6. Review the code
  7. Merge changes

Setting up Git

  • Download from git-scm.com
  • Install with default options
  • Open Git Bash or Command Prompt

$ git --version

Configuring Git

# Tell Git who you are

$ git config --global user.name "Your Name"

$ git config --global user.email "your.email@example.com"

Creating a GitHub Account

  1. Go to github.com
  2. Click "Sign Up"
  3. Follow the instructions
  4. Verify your email

Creating a Repository

New repo

$ mkdir my-project

$ cd my-project

$ git init

$ echo "# My Project" > README.md

$ git add README.md

$ git commit -m "First commit"

Clone existing repo

$ git clone https://github.com/username/repository.git

Pushing to GitHub

# Connect to GitHub

$ git remote add origin https://github.com/username/my-project.git

# Push your code

$ git push -u origin main

GitHub: Your Portfolio

  • Showcase your projects
  • Prove your coding skills
  • Build your reputation
  • Connect with other developers

Your coding resume!

Fork

A copy of someone else's repository on your account

  • Make your own copy of a project
  • Experiment with changes
  • First step to contribute to open source

Pull Requests (PR)

Asking to merge your changes

  • Notifies project owners
  • Shows your changes
  • Allows for code review
  • How contributions happen

Forking a Repository

  1. Find a repository on GitHub
  2. Click "Fork" button
  3. Select your account

# Clone your fork

$ git clone https://github.com/your-username/repository.git

Making Changes

# Create a branch

$ git checkout -b my-feature

# Edit files...

# Save changes

$ git add .

$ git commit -m "Add new feature"

# Push to GitHub

$ git push origin my-feature

Creating a Pull Request

  1. Go to your fork on GitHub
  2. Click "Compare & pull request"
  3. Add title and description
  4. Click "Create pull request"

Accepting a Pull Request

  1. Review the changes
  2. Add comments if needed
  3. Click "Merge pull request"
  4. Confirm the merge
